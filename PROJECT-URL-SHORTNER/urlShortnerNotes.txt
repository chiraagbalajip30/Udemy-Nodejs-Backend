Bootstrapping a Clean Express Server Setup for Backend Development


So, let us initialise the project by entering : "pnpm init" in the terminal. This will give you an "package.json" file. 


Then, we will install these as Dev Dependency:  pnpm i @types/node @types/express@4.x -D. Here we get all the node modules, etc. 


Then we will install express as a dependency: "pnpm i express@4.x".


Now, we will get the express server setup that is, we will make the server up and running. 
For that, create an index.js file. 

const PORT = process.env.PORT ?? 8000; - This is called as "Nullish Cohesion". 

import express from "express";

const app = express;

const PORT = process.env.PORT ?? 8000;

app.get("?", (req, res) => {
    return res.json({ status: 'Server is up and running..'});
});

app.listen(PORT, () => {
    console.log(`Server is running on ${PORT}`);
});

Configuring PostgreSQL with Docker for Local Development Environment


Step - 1  -  Setup Docker
----------------------------
Now, we will setup postgres with docker. 

So first create a file named "docker-compose.yml". 
(We are using this because we don't want to install postgreSQL locally in our machine. Instead, dockercreates an image of postgres, and a container for this inside that to let us build).


services:
  db:
    image: postgres
    restart: always
    environment:
      POSTGRES_USER: postgres
      POSTGRES_PASSWORD: admin
      POSTGRES_DB: postgres
    ports:
      - 5432:5432
    volumes:
      - db_data:/var/lib/postgresql/data

volumes:
  db_data:

🔹 Simple analogy

Think of it like this:

Container = rented room 🏠
Volume = locker 🔐

If you leave the room:
👉 Your stuff in the locker is still safe


🔹 When to use volumes

Use volumes when:

Database (PostgreSQL, MongoDB) ✅
File uploads (images, PDFs) ✅
Logs or backups ✅
🔹 Final answer

👉 We use volumes to persist data even after container restarts or deletion
👉 Without volumes, all data inside containers is temporary

Installing and Setting Up Drizzle ORM for Database


Step-2  -   Install node-postgres package
--------------------------------------------
Now let's get the Drizzle up and running. 


Look at the documention of Dirzzle ORm -> PostgreSQL.

Step-1
--
Instal these:
pnpm add drizzle-orm pg dotenv
pnpm add -D drizzle-kit tsx @types/pg


Step-3  -  Setup connection variables
------------------------------------------
Create .env file and place the database_url within it.


Step - 4  -  Connect Drizzle ORM to the database
-------------------------------------------------
Create a index.js file in the src directory and initialize the connection:

TRhis file is used for connection purpose between drizzle orm and the database. 

import "dotenv/config";
import { drizzle } from "drizzle-orm/node-postgres";

export const db = drizzle(process.env.DATABASE_URL);
export default db;


Step - 5  -  Create a table
------------------------------
Create a schema.ts file in the src/db directory and declare your table:


Create models folder, and add the tables or models howmuch ever there are that is users, bboks, etc. 

users.table.js code:

import { integer, pgTable, varchar, uuid } from "drizzle-orm/pg-core";

export const usersTable = pgTable("users", {
  id: uuid().primaryKey().defaultRandom(),
});


We have only added minimal code as of now and will be added later looking at the requirements. 




Step - 6  -  Setup Drizzle config file
------------------------------
Drizzle config - a configuration file that is used by Drizzle Kit and contains all the information about your database connection, migration folder and schema files.

Create a drizzle.config.js file in the root of your project and add the following content:


Create a index.js file inside models, and import usersTable here:

export {usersTable} from "./user.model";



Inside drzzle.config.js file, add this looking at the docs, but change the paths:

import 'dotenv/config';
import { defineConfig } from 'drizzle-kit';

export default defineConfig({
  out: './drizzle',
  schema: './models/index.js',
  dialect: 'postgresql',
  dbCredentials: {
    url: process.env.DATABASE_URL,
  },
});
