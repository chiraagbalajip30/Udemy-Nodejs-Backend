Bootstrapping a Clean Express Server Setup for Backend Development


So, let us initialise the project by entering : "pnpm init" in the terminal. This will give you an "package.json" file. 


Then, we will install these as Dev Dependency:  pnpm i @types/node @types/express@4.x -D. Here we get all the node modules, etc. 


Then we will install express as a dependency: "pnpm i express@4.x".


Now, we will get the express server setup that is, we will make the server up and running. 
For that, create an index.js file. 

const PORT = process.env.PORT ?? 8000; - This is called as "Nullish Cohesion". 

import express from "express";

const app = express;

const PORT = process.env.PORT ?? 8000;

app.get("?", (req, res) => {
    return res.json({ status: 'Server is up and running..'});
});

app.listen(PORT, () => {
    console.log(`Server is running on ${PORT}`);
});


The route is working as well when check through the postman.

Configuring PostgreSQL with Docker for Local Development Environment


Step - 1:
---------
Now, we will setup postgres with docker. 

So first create a file named "docker-compose.yml". 
(We are using this because we don't want to install postgreSQL locally in our machine. Instead, dockercreates an image of postgres, and a container for this inside that to let us build).


services:
  db:
    image: postgres
    restart: always
    environment:
      POSTGRES_USER: postgres
      POSTGRES_PASSWORD: admin
      POSTGRES_DB: postgres
    ports:
      - 5432:5432
    volumes:
      - db_data:/var/lib/postgresql/data

volumes:
  db_data:


We can also mount a "volume' so that the data is persistent.

Chatgpt:

🔹 Simple analogy

Think of it like this:

Container = rented room 🏠
Volume = locker 🔐

If you leave the room:
👉 Your stuff in the locker is still safe


🔹 When to use volumes

Use volumes when:

Database (PostgreSQL, MongoDB) ✅
File uploads (images, PDFs) ✅
Logs or backups ✅
🔹 Final answer

👉 We use volumes to persist data even after container restarts or deletion
👉 Without volumes, all data inside containers is temporary
